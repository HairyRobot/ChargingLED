I like the app, but it does not work on my device, can you add support for my device?

No. Reasons below.

- The app uses only standard API to detect charging cable plugged in, and to control the LED. No hack or special method is used.
- The app is hardware dependent. There are many Android devices, this developer possess just one of them.

Top
The app not working on my device.

There are many possible reasons, see below. (list is NOT exhaustive)

(1) Many devices have inbuild charging led flashing function that cannot be override using standard API. Some devices do allows users to disable it via Settings, for example the Samsung Galaxy S3, check you device to confirm.

(2) Android 4.2 Jelly Bean introduced the "Daydream" screen saver that appears when a device is either docked or charging. It conflict with the app, if you want ChargingLED, then disable the screen saver in Settings.
Note:
- Comments in Play Store reported "Daydream" does not conflict with ChargingLED.

(3) System (Android) is not detecting the "ACTION_POWER_CONNECTED broadcast".
To check this, enable "Debug mode" in app's settings, once enabled a toast "Power Connected" message is displayed whenever the charging cable is plugged in. If no message is displayed, and you confirmed there is no software conflict, then this mean the system is not detecting "ACTION_POWER_CONNECTED broadcast" which the app depends on. Your best action is to email your device's manufacturer, and see if they will release a firmware update to fix the problem.

(4) There is a software conflict with other installed apps. Check your installed apps for app that like ChargingLED also manipulate the system upon receiving "ACTION_POWER_CONNECTED broadcast". Once found, disable or uninstall those app in conflict.
The process is more optimistic if someone reported ChargingLED does works on your device, otherwise, your effort may just confirm it does not works on your device. Whatever the result, it would be helpful to other users to post your finding in Play Store comments (stating the device and os version).
Note:
- There is no easy way to determine the conflicting app, except checking each combination.
- Example of conflicting apps: any apps that also try to control the LED. e.g. Light Flow.

(5) The device itself may have built in charging LED indicator option, and is in conflict with this app.
For example, GS3 got this option located at "Settings > LED indicator > Charging".

(6) On some devices, the LED flashing don't start until the screen is off and the charge level changed since screen is off.
For example, GS4 (LineageOS).

Important Note:
- Hardware very from device to device, it is not possible to support all devices. Not even app like Light Flow works on all devices.
- ChargingLED is written using standard API, nothing special is used in the coding, if it doesn't works on your device, there is really not much I can do.
- See FAQ 10, 11, 12, 17 & 18 for more information.

I want to input the exact LED colour code, instead of using the colour picker.

There is no dedicated function for this, but advanced users can edit the exported config file "ChargingLED.ini", then import the new config back into the app to achieve this.

Note:
- The config file is commented to easily identify each charge level, anyone still not sure how to make the necessary change is advised to use the colour picker.

The "always lit" option don't works.

The default settings for "always lit" is 0/1 ms (off/on).
On some device (e.g. Nexus 5) this don't play well.
You can play with the blink settings to try to simulate the best settings for your device.

e.g. (off/on)
1) 0/0 - Strange, this actually produce "always lit" on Galaxy Nexus.
2) 0/30000
3) Your combination here.

Note:
- If you can't achieve the desire "always lit" result, there is really not much I can do, as blinking/lit is defined by those two parameters.
- The hardware will not keep the LED "always lit" if it does not support this.
- To enter ms outside those provided, you can do it via the exported "ChargingLED.ini" config file.

What are the different Mode.

"LED" - flash LED according to charging status
"Notification" - display charging status notification on the system status bar
"Sound" - sound ringtone (notification) on reaching defined charging status

Note:
- "Notification" and "Sound" maybe useful on devices that does not have LED or LED don't fully works.
- Enable at least one mode, otherwise, app will be disabled.

Top
The blink interval does not match the specified ms.

ChargingLED will call the corresponding API with the specified ms, but whether the LED will blink for the specified ms is dependent on the hardware.

Note:
- The hardware cannot blink the LED longer then it can support.
- The hardware cannot keep the LED "always lit" if it does not support this.

Why is the LED only blink while screen is off.

The LED blink only while screen is off is a system constraint.
ChargingLED does not programmatically switch off the LED while screen is on.

I get two different ringtone on plugging or unplugging the charging cable.

The additional ringtone is likely generated by Android, check "Charging sounds" option at "Settings > Sound & notification > Other sounds".

Decide if ChargingLED or Android should be responsible for playing the ringtone, and enable/disable accordingly.

Note:
- Above system settings is introduced in Android 6.0 (Marshmallow), and may differ slightly in newer Android, so just look for something similar.

I not getting any notification (LED/Status/Sound) from the ChargingLED app.

Check if notification is blocked by the system.

Goto "Settings > Sound & notification > App notifications", then select "ChargingLED", then check notification from ChargingLED is not blocked/disabled.

Note:
- Above system settings is introduced in Android 6.0 (Marshmallow), and may differ slightly in newer Android, so just look for something similar.

The sound notification does not repeat.

The sound notification is repeated only if during charging the battery level is maintained at or above the defined "Battery level" setting.
The scheduled repeat sound notification will get cancel, if the battery level drop below the defined "Battery level" setting.

For some reason, on my Galaxy Nexus, the device fully charge the battery to 100% as reported by Android, on continue charging, Android would then report 99%, and sometime may not report 100% again, so to ensure I get the repeat sound notification, I set the "Battery level" to 99%.

Note:
- Enable "Charging notification" to see the current battery level as reported by Android.

Why do I get the "Unplug sound" upon plugging in the charge cable?

The "Unplug sound" is played upon the app receiving the unplugged signal from Android.

You can confirm this by setting the app to debug mode, which if enabled, will display a toast message upon receiving the following signal from Android:

- a charge cable plugged in
- a charge cable unplugged

Therefore, the likely reason is following events happened:

(1) Plugin the charge cable
(2) the app received a charge cable plugged in signal from Android. (Play sound as appropriate)
(3) the app received a charge cable unplugged signal from Android. (Play sound as appropriate)
(4) the app received a charge cable plugged in signal from Android. (Play sound as appropriate)

As to why (3) and (4) happened, that is a question for Android.

Note:
- Similar would happen if "Plugin sound" is enabled.

I enabled [LED(on), Notification(off), Sound(off)], but still see notification on the status bar?

After the release of v0.3.0, users started reporting app fc when LED is enabled on some devices.

After some investigation, the problem maybe related to the changes on the notification API, and me bumping minSdkVersion to 15 from 7 may have broken the app on some devices, those apparently not my test devices.

I believe Play Store only allow app to more forward, i.e. once an app support a higher API, it can't go back and not support it, because Play Store will not accept the new apk file.

The remedy to the situation is to have notification on the status bar when LED is enabled.

Top
I selected a ringtone, but the app show "???".

The "???" indicate a fc caused by SecurityException occurred.
This happen when the selected ringtone is from external storage and the app is not granted the READ_EXTERNAL_STORAGE permission. As READ_EXTERNAL_STORAGE permission is requested in manifest, this affect devices on API >= 23 (Marshmallow).

To remedy the situation, either

(1) do a settings export, this will prompt user to grant the require storage permission.
(2) goto the ChargingLED app info screen and manually grant it storage permission.

Why is there two ChargingLED icon on the notification status bar?

As of Android 8.0 (Oreo), if an app is targeted API-26 or above, a service needs to run in the foreground, to achieve this, a foreground notification is needed, hence the extra ChargingLED icon on the notification status bar.

To stop the foreground notification from showing on the status bar, follow below steps:
(1) goto the App info screen of ChargingLED.
(2) tap on "App notifications".
(3) switch off the "Foreground service status" notification.

Note:
- If you have apps killer running on your device, ensure to exclude ChargingLED from its kill list.

What is Doze Mode?

In Android 6.0 (Marshmallow), the Doze Mode was introduced to prevents certain tasks from running if your device is in idle state, this is to reduce power consumption by deferring background CPU and network activity for applications.

This may cause ChargingLED to stop working, because the app uses some of the Doze mode restricted functions (wake lock, AlarmManager, JobScheduler).
If app is affected, consider excluding the app from battery optimization to allows timely execution of required functions.

Follow below steps to exclude app from optimization:
(1) goto the App info screen of ChargingLED.
(2) tap on "Battery" to goto the Battery usage screen.
(3) tap on "Battery optimization" to goto the Battery optimization screen.
(4) filter for "All apps".
(5) locate ChargingLED, then tap it to select.
(6) select "Don't optimize".

or from System settings with below steps:
(1) goto System settings.
(2) tap on "Battery" to goto Battery screen.
(3) select "Battery optimization" from the menu bar to goto the Battery optimization screen.
(4) filter for "All apps".
(5) locate ChargingLED, then tap it to select.
(6) select "Don't optimize".

Note:
- ChargingLED only run during the device is charging, once the charging cable is unplugged, the app terminate, including any background service.

Why is there a delay in detecting charging cable plugged in?

On Android 8.0 (Oreo) or above, due to "Background execution limits" introduced to improve battery life, Android no longer register broadcast receivers for implicit broadcasts in the manifest, this mean ChargingLED may take up to 3-4 minutes before receiving the signal from Android that it detected USB cable plugged in for charging.

This delay would affect the "Plugin sound" function, others functions should function as normal once charging is detected.

Note:
- This does not affect any apps that detect "ACTION_POWER_CONNECTED" in a service, since ChargingLED is designed specifically to avoid a constantly running service, it is something that can't be avoided.
